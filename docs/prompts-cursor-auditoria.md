# Prompts Cursor - Auditoría

## Prompt 1 - Auditoría del repositorio actual

Usa `/docs/requerimientos-portal-empresa.md` como fuente de verdad funcional para reconstruir esta demo.

Objetivo:
convertir el prototipo actual en una demo navegable de Portal de Proveedores para EMPRESA, basada en flujos de negocio y navegación por rol.

Primera tarea:
analiza el repositorio actual y genera una auditoría funcional y técnica comparando el estado actual contra `/docs/requerimientos-portal-empresa.md`.

Entrega en `/docs`:
1. `auditoria-demo-actual.md`
2. `matriz-cobertura-actual.md`
3. `plan-reconstruccion.md`

La auditoría debe incluir:
- estructura actual del proyecto
- mapa de pantallas existentes
- evaluación de la navegación actual
- pantallas conectadas vs pantallas aisladas
- evaluación por requerimiento
- clasificación: conservar / adaptar / rehacer / eliminar
- propuesta de reconstrucción por fases

No empieces a reescribir toda la app todavía.
Primero entrega el diagnóstico y una propuesta clara de sitemap, rutas y estrategia.
